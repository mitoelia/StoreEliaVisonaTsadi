Requisiti di sistema per il driver JDBC

Per accedere ai dati di un database di SQL Server tramite Microsoft JDBC Driver per SQL Server, è necessario che nel computer siano installati i componenti seguenti:

  • Microsoft JDBC Driver per SQL Server

  • Java Runtime Environment

Requisiti di Java Runtime Environment

A partire da Microsoft JDBC Driver per SQL Server, il supporto del driver JDBC per l'API della specifica Java Database Connectivity (JDBC) è stato esteso in modo da includere l'API di JDBC 4.0. JDBC 4.0 è stato introdotto come parte di Sun Java SE Development Kit (JDK) 6.0 e Java Runtime Environment (JRE) 6.0. JDBC 4.0 è un superset di JDBC 3.0.

Microsoft JDBC Driver per SQL Server supporta sia JDBC 3.0 sia JDBC 4.0.

Quando si distribuisce Microsoft JDBC Driver per SQL Server in sistemi operativi Windows e UNIX, è necessario utilizzare rispettivamente i pacchetti di installazione sqljdbc_<versione>_enu.exe e sqljdbc_<versione>_enu.tar.gz. Per ulteriori informazioni su come distribuire il driver JDBC, vedere Distribuzione del driver JDBC.

Per supportare la compatibilità con le versioni precedenti e i possibili scenari di aggiornamento, il driver JDBC include due librerie di classi JAR in ogni pacchetto di installazione, ovvero sqljdbc.jar e sqljdbc4.jar.

JAR

Descrizione

sqljdbc.jar

La libreria di classi sqljdbc.jar fornisce il supporto per JDBC 3.0.

La libreria di classi sqljdbc.jar richiede la versione 5.0 di Java Runtime Environment (JRE). Se si utilizza sqljdbc.jar in JRE 6.0, verrà generata un'eccezione quando si stabilisce la connessione a un database.

Il driver JDBC non supporta JRE 1.4. Quando si utilizza il driver JDBC, è necessario aggiornare JRE 1.4 a JRE 5.0 o a JRE 6.0. In alcuni casi potrebbe essere necessario ricompilare l'applicazione perché potrebbe non essere compatibile con JDK 5.0 o versioni successive. Per ulteriori informazioni, vedere la documentazione disponibile nel sito Web Sun Microsystems.

sqljdbc4.jar

La libreria di classi sqljdbc4.jar fornisce il supporto per JDBC 4.0. Include tutte le funzionalità di sqljdbc.jar, nonché i nuovi metodi di JDBC 4.0.

La libreria di classi sqljdbc4.jar richiede la versione 6.0 di Java Runtime Environment (JRE). Se si utilizza sqljdbc4.jar in JRE 1.4 o 5.0, verrà generata un'eccezione.

Se l'applicazione deve essere eseguita in JRE 6.0, utilizzare sqljdbc4.jar anche se nell'applicazione non vengono utilizzate funzionalità di JDBC 4.0.

Si noti che il driver JDBC è stato progettato per l'utilizzo e il supporto da parte di tutte le principali macchine virtuali Java equivalenti a quelle Sun, ma è stato testato solo su Sun JRE 5.0 o versione successiva.

Requisiti di SQL Server

Il driver JDBC supporta le connessioni a SQL Server 2005 e versioni successive.

Requisiti del sistema operativo

Il driver JDBC è stato progettato per l'esecuzione in qualsiasi sistema operativo che supporti l'utilizzo di Java Virtual Machine (JVM). Ufficialmente, tuttavia, sono stati testati solo i sistemi operativi Sun Solaris, SUSE Linux, Windows Vista e Windows 7.

Lingue supportate

Il driver JDBC supporta tutte le regole di confronto delle colonne di SQL Server. Per ulteriori informazioni sulle regole di confronto supportate dal driver JDBC, vedere Caratteristiche internazionali del driver JDBC.

Per ulteriori informazioni sulle regole, vedere "Utilizzo delle regole di confronto" nella Documentazione online di SQL Server.
