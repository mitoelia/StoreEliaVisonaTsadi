Novità nel driver JDBC

In Microsoft JDBC Driver 4.0 per SQL Server sono inclusi diversi nuovi miglioramenti e funzionalità.

Informazioni sulla connessione a un database di SQL Azure

È disponibile un nuovo argomento con informazioni sulla connessione a un database di SQL Azure. Per ulteriori informazioni, vedere Connessione a un database in SQL Azure.

Supporto per il ripristino di emergenza a disponibilità elevata

Microsoft JDBC Driver 4.0 per SQL Server supporta connessioni di ripristino di emergenza a disponibilità elevata a Gruppi di disponibilità AlwaysOn in SQL Server 2012. Per ulteriori informazioni, vedere Supporto di Microsoft JDBC Driver per il ripristino di emergenza a disponibilità elevata.

Utilizzo dell'autenticazione integrata Kerberos per la connessione a SQL Server

In Microsoft JDBC Driver 4.0 per SQL Server è disponibile l'autenticazione integrata Kerberos di tipo 4 per la connessione a un database di SQL Server. Per ulteriori informazioni, vedere Utilizzo dell'autenticazione integrata Kerberos per la connessione a SQL Server. L'autenticazione integrata Kerberos di tipo 2 è disponibile nelle versioni di Microsoft JDBC Driver per SQL Server precedenti alla 4.0.

Accesso a informazioni di diagnostica nel log degli eventi estesi

È possibile accedere al contenuto del log degli eventi estesi per ottenere informazioni sugli errori di connessione. Per ulteriori informazioni, vedere Accesso a informazioni di diagnostica nel log degli eventi estesi.

Supporto aggiuntivo per le colonne di tipo sparse

Se nell'applicazione è già stato eseguito l'accesso ai dati di una tabella in cui vengono utilizzate colonne di tipo sparse, si sarà notato un aumento delle prestazioni. È possibile ottenere informazioni sulle colonne, incluse le informazioni sulle colonne di tipo sparse, con Metodo getColumns (SQLServerDatabaseMetaData). Per ulteriori informazioni sulle colonne di tipo sparse in SQL Server, vedere Utilizzo di colonne di tipo sparse.

Il metodo isSparseColumnSet viene aggiunto alla classe SQLServerResultSetMetaData per indicare se una colonna in un set di risultati è un set di colonne di tipo sparse.

Xid.getFormatId

Prima di Microsoft JDBC Driver 4.0 per SQL Server il driver JDBC riceveva un identificatore di formato dall'applicazione ma non passava tale valore al server di database. Passava invece il valore 0.

A partire da Microsoft JDBC Driver 4.0 per SQL Server il driver JDBC passa l'identificatore di formato dall'applicazione al server di database. Per ottenere il comportamento aggiornato, verificare che il file sqljdbc_xa.dll nel server sia aggiornato. Per ulteriori informazioni sulla copia di una versione aggiornata del file sqljdbc_xa.dll nel server, vedere Informazioni sulle transazioni XA.

Itanium non supportato per le applicazioni JDBC Driver 4.0

L'esecuzione delle applicazioni Microsoft JDBC Driver 4.0 per SQL Server in un computer Itanium non è supportata.
